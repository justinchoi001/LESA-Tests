LESA - Search - Advanced Search

Prerequisites:

  • Liferay CSE account

Description:

This test is designed to have users test the advanced search feature. Users will search for tickets using Account, Assigned to, Severity, Escalation Level, Resolution, Status, Component, and Due Date Range.

Road Map:

  1. Sign In As Liferay CSE
  2. Search By Account
  3. Search By Severity
  4. Search By Escalation
  5. Search By Assigned To
  6. Search By Resolution
  7. Search by Status
  8. Search By Component
  9. Search By Due Date Range

Test Steps:

  1. Sign in as a Liferay CSE
  2. Click the Advanced link (if Basic Search displays)
    Search by Account
  3. Enter "UMass" in the Account field
  4. Click the Search button
  5. Assert that only UMASS tickets display in the results section
  6. Click the Reset button
    Search by Severity
  7. Click the Critical check box
  8. Click the Search button
  9. Assert that only critical tickets display (red banner)
  10. Click the Reset button
    Search by Escalation
  11. Click the P1 check box
  12. Click the Search button
  13. Assert that only P1 tickets display in the results section
  14. Click the Reset button
    Search by Assigned To
  15. Click the Choose button next to "Assigned to"
  16. Click another Liferay CSE
  17. Click the Search button
  18. Assert that only tickets assigned to this CSE display
  19. Click the Reset button
    Search by Resolution
  20. Click the Duplicate check box
  21. Click the Search button
  22. Assert that all the tickets display "Closed" as their status
  23. Click on any ticket
  24. Assert that "Duplicate" display as their resolution
  25. Click Back to Previous Page
  26. Click the Reset button
    Search by Status
  27. Click the On Hold check box
  28. Click the Search button
  29. Assert that all the tickets display "On Hold" as their status
  30. Click the Reset button
    Search by Component
  31. Click the Clustering check box
  32. Click the Search button
  33. Click on the first three articles
  34. Assert that Component: Clustering displays in each Description box
  35. Click the Reset button
    Search by Due Date Range
  36. Select "May 1, 2014" in the Begin Date drop down under Due Between
  37. Select May 15, 2014: in the End Date drop down under Due Between
  38. Click the Search Button
  39. Assert that there are no tickets with due dates beyond May 15, 2014
  40. Click Last
  41. Assert that there are no tickets with due dates before May 1, 2014
